# Problem with functional programming
Gradually understanding and tracking large amounts of code becomes difficult.

# OOPS
Object-Oriented Programming (OOP) is a programming paradigm based on organizing software design around data, or "objects," rather than functions and logic

# Abstraction
The process of managing complexity by hiding, removing, or ignoring unnecessary implementation details to focus on essential, high-level features

# Encapsulation

Encapsulation in OOP is the bundling of data (attributes) and methods (functions) that operate on that data into a single unit, known as a class, while restricting direct access to some of an object's components

# Polymorphism

The ability of a single interface, function, or object to take on multiple forms, such as the addition sign which can be used for sum and also for concatenation

# Inheritance

A fundamental Object-Oriented Programming (OOP) mechanism where a new class (child/derived/subclass) inherits attributes and behaviors (fields/methods) from an existing class (parent/base/superclass).
